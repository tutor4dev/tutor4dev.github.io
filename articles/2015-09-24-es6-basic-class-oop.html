เขียน JavaScript แบบ OOP บน ECMAScript 6

ESMAScript 6 นั้นเป็นมาตรฐานที่มีมาซักระยะนึงแล้วก็จริง แต่การออก Released 4.x.x ของ Node.js นั้นก็ถือเป็นจุดเปลี่ยนที่สำคัญ เพราะทำให้เราสามารถใช้งาน Feature ของ ECMAScript 6 แบบ Out-of-the-Box โดยไม่ต้องพึ่งพาตัวช่วยอื่นๆ

ในบทความนี้ผู้เขียนจะของพูดถึง Feature ในด้าน OOP ซึ่งเป็น 1 ใน Feature ที่หลายๆ คนรอคอย เราลองมาทดสอบไปพร้อมๆ กันครับว่า การเขียน OOP ในเวอร์ชั่นของ JavaScript นั้นจะเป็นอย่างไรบ้าง

ตรวจสอบเวอร์ชั่นของ Node.js

พิมพ์คำสั่งที่ Terminal


node --version

เวอร์ชั่นของ Node.js ที่รองรับ Feature ของ ECMAScript 6 คือ 4.x.x

ประกาศ Class


"use strict";

class Person {

}

"use strict"; เป็นข้อบังคับในการใช้งาน class

Class Constructor


class Person {
    constructor(name = 'Anonymous') {
        this.name = name;
    }
}

ค่อนข้างจะเป็น Common Case ที่การสร้าง Instance หรือ Object ควรจะมี Constructor เพื่อทำการ Initialize ค่าของตัวแปรบางตัวที่มีความสำคัญ, ซึ่งใน ES6 สามารถกำหนด Default Value ได้ด้วยหากผู้ใช้ไม่ได้ทำการผ่านค่าตัวแปร

สร้าง Object


// ใช้ค่า default
var person = new Person();

// ผ่านค่าตัวแปร
var person = new Person('John');

Prototype Method


class Person {
    sayHello() {
        console.log('Hello');
    }

    sayName() {
        console.log('My name is ' + this.name);
    }
}

var person = new Person('John');
person.sayHello();
person.sayName();

Prototype Method หลายคนอาจจะคุ้นเคยในชื่อว่า Instance Method ซึ่งก็คือ Method ที่ถูกเรียกใช้โดย Instance ของ Class นั่นเอง

Static Method


class Person {
    static classDescription() {
        console.log('This class represents any persons');
    }
}

Person.classDescription();

Static Method ในบางภาษาโปรแกรมมิ่งจะเรียกว่า Class Method ซึ่งก็คือ Method ที่ถูกเรียกใช้โดยตัว Class ซึ่งการเรียกใช้ Static Method นั้น เราไม่มีความจำเป็นต้องทำการ สร้าง Instance หรือ Object ของ Class นั้นๆ

Getter และ Setter


class Person {
    get getName() {
        return this.name;
    }

    set setName(name) {
        if (name) {
            this.name = name;
        }
    }

    set setDetail(detail) {
        if (detail) {
            var detail = detail.toString().split(';');
            this.age = detail[0];
            this.gender = detail[1];
        }
    }
}

var person = new Person('John');
console.log(person.getName);
person.setName = 'Peter';
person.setDetail = '25;male';

Getter และ Setter ใน ES6 นั้นจะมีความพิศดารซักนิดหากเทียบกับภาษาโปรแกรมมิ่งตัวอื่นเพราะ แทนที่เราจะได้ Method เรากลับได้ Property ในการ Access ตัวแปร Internal ของ Instance หรือ Object แทน

การผ่าน Parameter กับ Setter นั้นสามารถมีได้เพียง 1 Parameter เท่านั้น หากเรามีความจำเป็นที่ต้องการใช้ Setter หลายๆ ตัวแปร เราจะต้องเลี่ยงมาใช้เทคนิค Split ค่า String แทน

การ Inheritance


class Student extends Person {
    constructor(name, studentId) {
        super(name);
        this.studentId = studentId;
    }

    sayHello() {
        super.sayHello();
        console.log('I am a student, my ID is ' + this.studentId);
    }
}

var student = new Student('Tom', 12345);
student.sayHello();

console.log(person instanceof Person);
console.log(student instanceof Person);
console.log(student instanceof Student);

เราสามารถทำการ Inherit จาก Class ใดๆ และ ทำการ Override Method ที่ต้องการโดยการใช้ super

บทสรุป

Feature ในเรื่อง Class และ OOP นั้นเป็น 1 ใน Feature ของ ECMAScript 6 ที่หลายๆ คนรอคอย เพราะการเขียนโปรแกรมแบบ OOP จะช่วยให้การเขียนโปรแกรมนั้นมีประสิทธิภาพเพิ่มขึ้นเยอะพอสมควรเลยทีเดียว ถึงแม้ OOP ใน ECMAScript 6 นั้น อาจจะยังไม่ได้ Strong และมี Feature ของ OOP ครบถ้วน เมื่อเทียบกับ OOP ในภาษาโปรแกรมมิ่งอื่นๆ แต่ก็ถือเป็นจุดเริ่มต้นที่ดี ที่จะทำให้เราสามารถใช้ JavaScript แบบ Pure ได้โดยไม่ต้องไปพึ่งตัวช่วยอื่นๆ เช่น TypeScript หรือ Transpiler อย่างเช่น Babel เป็นต้น
